## 2026-02-20 - npm Binary Distribution Research (sidebar_tui-8fqc)

Researched npm binary distribution for Rust CLIs and closed sidebar_tui-8fqc. Cloned esbuild, biome, and cargo-dist repos locally to `references/npm-binary-distribution/`. The recommended pattern is `optionalDependencies`: the main `sidebar-tui` package lists platform-specific packages (e.g. `sidebar-tui-darwin-arm64`) as optionalDependencies; npm/bun install only the one matching the current platform based on `os`/`cpu` fields; the main package's `bin/sb` launcher script detects the platform and spawns the correct binary via `require.resolve()`. The alternative (cargo-dist's postinstall download pattern) is simpler but slower and requires internet at install time. A comprehensive README with code examples, package.json templates, and launcher script template is at `references/npm-binary-distribution/README.md`. Key reference: `references/npm-binary-distribution/biome/` (Rust CLI, most directly applicable, includes the full GitHub Actions release workflow).

## 2026-02-20 - Self-Update and Version Check Research (sidebar_tui-baj8)

Researched the self-update mechanism for sidebar_tui, closing sidebar_tui-baj8. The key finding is that two crates should be used together: `update-informer` (for daily GitHub API version checks with file-mtime-based caching using XDG-compliant directories via the `etcetera` crate) and `self_update` (for actually downloading and replacing the binary from GitHub Releases). Cloned both crates plus `etcetera` to `references/self-update/`. For Homebrew detection, check `std::env::current_exe()` for `/homebrew/`, `/Homebrew/`, or `/linuxbrew/` in the path, or check the `HOMEBREW_PREFIX` env var — if Homebrew-installed, skip self-update and print the `brew upgrade sb` message instead. The XDG cache pattern (from `update-informer/src/version_file.rs`): write a file to `$XDG_CACHE_HOME/sidebar-tui/update-check`, check its mtime against 24h to decide whether to hit the GitHub API. A comprehensive `RESEARCH_NOTES.md` with all code examples is at `references/self-update/RESEARCH_NOTES.md`.

## 2026-02-20 - Homebrew Tap and AUR Packaging Research (sidebar_tui-vxom)

Completed research for Homebrew tap and AUR packaging for sidebar-tui. All references saved to `references/homebrew-aur/`. Key findings: for Homebrew, `cargo-dist` is the recommended tool — it generates per-arch binary formulas and pushes them to a `homebrew-tap` repo on release; the `mislav/bump-homebrew-formula-action` cannot handle multi-arch binary formulas with if/else arch-specific URLs. Best reference examples: `cargo-dist-tap-formula-example.rb` (homebrew formula pattern), `aur-gitui-bin-PKGBUILD` (multi-arch AUR), and `aur-deploy-workflow-sane-fmt.yml` (AUR GitHub Actions workflow). Full decision guide in `references/homebrew-aur/README.md`. Updated bd issue sidebar_tui-vxom with notes.

## 2026-02-20 - Implemented Windows TCP Socket Support for Daemon IPC (sidebar_tui-ck6z)

Implemented platform-specific IPC as specified: on Unix, the daemon still uses Unix domain sockets (`daemon.sock`); on Windows, it binds a `TcpListener` to `127.0.0.1:0` (OS assigns the port), writes the port to a lockfile (`daemon.port`), and clients read the lockfile to connect. Added a `pub type IpcStream` alias (`UnixStream` on Unix, `TcpStream` on Windows) exported from `daemon.rs`, imported in `main.rs`. Moved stream setup (set_nonblocking + set_read_timeout) out of `handle_client` into the `run_platform()` accept loop so `handle_client` takes the concrete `IpcStream`. All function signatures in `main.rs` that previously took `&mut UnixStream` were updated to `&mut IpcStream`. Added `platform_default_runtime_dir()` helper so `get_runtime_dir()` works on Windows (uses `%LOCALAPPDATA%`). All 414 lib tests pass. Closed sidebar_tui-ck6z.
