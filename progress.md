## 2026-02-15 - Critical Crash Bug Fixed (Enter to Select Session)

Fixed sidebar_tui-3wl: The TUI was crashing with "Broken pipe (os error 32)" when pressing Enter in the sidebar to select a session. Root cause: daemon.rs line 1057 was breaking the client handler loop after receiving `ClientMessage::Detach`, closing the connection. But the client in main.rs was trying to reuse the same connection to send `ClientMessage::Attach` for session switching. Fix: Removed `DaemonResponse::Detached` from the break condition in handle_client() - now only `ShuttingDown` breaks the loop. This allows the client to detach from one session and attach to another on the same connection. Added E2E test `test_session_selection_no_crash` that creates two sessions and verifies switching between them doesn't crash. All 73 tests pass (54 lib/bin + 17 E2E + 2 scaffold). No clippy warnings. Binary reinstalled. Closed sidebar_tui-3wl. Next agent should pick up one of: sidebar_tui-qd0 (color change), sidebar_tui-7ex (sidebar padding), or another P2 issue.
